## 画面 UI の具体イメージ

### 1. 折れ線グラフ（メンション返信の件数推移）

期間内の「1日あたりフィードバック件数」の推移。

```
件数
 3 │         ●
 2 │    ●         ●
 1 │         ──────
 0 └─────────────────
   4/29 4/30 5/1  5/2  5/3  5/4
```

- X軸: 日付
- Y軸: その日の `feedback_received` イベント件数
- 用途: 「この週はよくフィードバックした / 止まっている」の傾向把握

### 2. Pie チャート（learned_preferences 更新頻度）

フィードバックのうち、Claude が実際に「好み」を抽出して学習に反映できた割合。

```
設定更新あり 70%  ← learned_preferences_updated = true
設定更新なし 30%  ← 具体的な好みを抽出できなかった
```

- 用途: F8 学習機構が実際に機能しているか確認する指標

### 3. 一覧テーブル（フィードバック履歴）

| 受信日時 | 実行ID | フィードバック内容（先頭200字） | 設定更新 | 新規設定数 |
|---|---|---|---|---|
| 2026-05-04 14:30 | exec-abc… (リンク) | 「もう少し詳しく説明して...」 | ✅ | 2 |
| 2026-05-03 09:15 | exec-def… (リンク) | 「わかりやすかったです」 | ❌ | 0 |

- 実行IDをクリック → 実行詳細画面（イベントタイムライン）へ遷移
- 用途: どのトピックへのフィードバックが学習に反映されたか追跡

### 画面全体のレイアウト

```
┌─ フィードバック分析 ─────────────── [7d ▼] ─┐
│ [KPI: 総数] [KPI: 更新件数] [KPI: 平均新規] [KPI: 累計] │
│                                                          │
│ ┌── 件数推移（折れ線）────────────────────────────────┐ │
│ │  (日別カウントグラフ)                               │ │
│ └────────────────────────────────────────────────────┘ │
│                                                          │
│ ┌── 設定更新率（Pie）──┐ ┌── 絵文字反応（将来実装予定）┐ │
│ │  (更新あり/なし)     │ │  データ未収集               │ │
│ └──────────────────────┘ └─────────────────────────────┘ │
│                                                          │
│ ┌── フィードバック履歴（テーブル）─────────────────────┐ │
│ │  日時 / 実行ID / 内容 / 更新 / 設定数               │ │
│ └────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────┘
```
